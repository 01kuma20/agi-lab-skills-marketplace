# Career Compass — スキルギャップ分析

プロフィールと求人URLを元に分析を実行します。

**前提：** 呼び出し元からプロフィールJSONが渡されていること。

---

## ステップ 1：求人URL入力

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🔗 分析したい求人のURLを貼り付けてください
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

URLが入力されたら WebFetch でHTMLを取得し、以下を抽出する：

- **job_title**：求人タイトル
- **company**：会社名
- **must_skills**：必須スキル・経験の一覧
- **want_skills**：歓迎スキル・経験の一覧
- **job_description**：業務内容の要約（3-5行）
- **salary**：想定年収・待遇（なければ空文字）

> HTMLが長い場合は `<meta>`・`<title>`・`<h1〜h3>`・スキル記載箇所を優先的に解析する。

---

## ステップ 2：追加ヒアリング

`must_skills` をプロフィールの `skills.technical` + `skills.business` + `work_history[].tech_stack` と照合する。

**プロフィールから確認できない必須スキルが1つでもあれば**、1問ずつ確認する：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🔍 追加確認（求人必須要件）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

この求人では「<スキル名>」が必須とされていますが、
ご経験はありますか？（ある場合は概要も教えてください）
```

- `want_skills` は確認しない
- プロフィールに既にあるスキルは聞かない
- 「ある」と答えた場合はその内容を分析に反映する

---

## ステップ 3：スキルギャップ分析

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📊 スキルギャップ分析
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【求人】<job_title> @ <company>

✅ マッチスキル（充足）
  - スキル1

⚠️  ギャップスキル（要強化）
  - スキルA（必須）
  - スキルB（歓迎）

マッチスコア：
```

スコア算出後にプログレスバーを表示：

```bash
bash "${CLAUDE_PLUGIN_ROOT}/scripts/progress_bar.sh" <スコア0-100>
```

> スコア算出式：`(マッチした必須スキル数 × 2 + マッチした歓迎スキル数) / (全必須スキル数 × 2 + 全歓迎スキル数) × 100`
> 分母が0の場合はスコア50。

---

## ステップ 4：学習ロードマップ生成

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🗺️  学習ロードマップ
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【優先度：高】
  🔴 <スキルA>
     習得目安：<期間>
     📚 書籍：「<書籍名>」（著者）
     🎥 講座：<Udemy/Coursera等のコース名>
     🛠️  ハンズオン：<具体的な実践方法>

【優先度：中】
  🟡 <スキルB> ...

【優先度：低】
  🟢 <スキルC> ...
```

> 優先度：必須ギャップ → 高、歓迎ギャップ → 中/低（経験年数で調整）
